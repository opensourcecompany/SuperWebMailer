Statusreport an Ihre E-Mail-Adresse <email-adresse> senden Sendet nach Abschluss des E-Mail-Versands eine E-Mail mit dem Statusreport an die angegebene E-Mail-Adresse. Die E-Mail-Adresse wird aus den hinterlegten Benutzerdaten übernommen.

 
Statusreport an den Administrator der Empfängerliste senden Sendet nach Abschluss des E-Mail-Versands eine E-Mail mit dem Statusreport an den Administrator/Ersteller der Empfängerliste.

 
Statusreport an alle zugewiesenen Nutzer der Empfängerliste senden Sendet nach Abschluss des E-Mail-Versands eine E-Mail mit dem Statusreport an alle Nutzer der Empfängerliste, die Zugriff auf die Empfängerliste haben.

 
Statusreport an folgende E-Mail-Adresse senden Sendet nach Abschluss des E-Mail-Versands eine E-Mail mit dem Statusreport an die angegebene E-Mail-Adresse.

 
E-Mailing nur speichern Speichert nur das E-Mailing. Sie können später nochmals die Einstellung des E-Mailings ändern und eine andere Option wählen, um den E-Mail-Versand anzustoßen.

Hinweis:
Steht nur diese Variante zur Verfügung, dann muss ein Administrator oder berechtigter Nutzer das E-Mailing überprüfen und die korrekte Planung für den Versand einstellen.

 
E-Mailing sofort live versenden Verwenden Sie diese Versandvariante um das E-Mailing/Newsletter nach Durchlaufen des Assistenten im aktuell geöffneten Browserfenster versenden zu lassen.

Hinweis:
Es muss eine Internetverbindung dauerhaft bestehen, der Browser darf während des E-Mail-Versands nicht geschlossen werden.

 
E-Mailing nach Speicherung der Einstellungen versenden (CronJob) Verwenden Sie diese Versandvariante um das E-Mailing/Newsletter nach Durchlaufen des Assistenten durch das CronJob-Script sofort im Hintergrund versenden zu lassen.

Hinweis:
Das CronJob-Script muss korrekt installiert worden sein, so dass es regelmäßig ausgeführt wird. Während des E-Mail-Versands muss kein Browser geöffnet sein, da der Server selbständig die E-Mails versendet.

Die Versandzeit des E-Mailings kann beschränkt werden, z.B. Versand jeden Tag zwischen 8 und 17 Uhr. Bei einer hohen Anzahl E-Mail-Empfängern oder gewollten verlangsamten Versands, z.B. zum Schutz der E-Mail-Reputation des E-Mail-Servers, kann so sicher gestellt werden, dass die Empfänger tagsüber die E-Mail erhalten.

 
E-Mailing zu einem späteren Zeitpunkt versenden (CronJob) Aktivieren Sie diese Versandvariante und wählen Sie den Versandzeitpunkt für den einmaligen zeitversetzten E-Mail-Versand.

Hinweis:
Das CronJob-Script muss korrekt installiert worden sein, so dass es regelmäßig ausgeführt wird. Während des E-Mail-Versands muss kein Browser geöffnet sein, da der Server selbständig die E-Mails versendet.

Bei einer hohen Anzahl E-Mail-Empfängern oder gewollten verlangsamten Versands, z.B. zum Schutz der E-Mail-Reputation des E-Mail-Servers, kann so sicher gestellt werden, dass die Empfänger tagsüber die E-Mail erhalten.

 
E-Mailing mehrfach versenden (CronJob) Verwenden Sie diese Versandvariante um das E-Mailing mehrfach versenden zu lassen. Geben Sie dazu die Uhrzeit an und wählen Sie Tag oder Wochentag und Monat aus.

Hinweise:

  • Das CronJob-Script muss korrekt installiert worden sein, so dass es regelmäßig ausgeführt wird. Während des E-Mail-Versands muss kein Browser geöffnet sein, da der Server selbständig die E-Mails versendet.
  • Legen Sie die Uhrzeit für den E-Mail-Versand so fest, so dass der Versand an alle E-Mail-Empfänger möglich ist. Planen Sie z.B. einen großen E-Mail-Versand für einen Mittwoch um 23:59 Uhr und das Script schafft die Abarbeitung der zu versendenden nicht, dann könnte es sein, das nur ein Teil der E-Mails versendet wird, da der nächste Tag ein Donnerstag ist.
  • Besitzt ein Monat nur 30 Tage und Sie planen den Versand für den 31. jeden Monats, dann werden in den Monaten mit weniger als 31 Tagen keine E-Mails versendet.
